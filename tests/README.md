# tests/ — 自動テスト

## レイアウト

```
tests/
├── core/        # コア層ユニットテスト（カバレッジ 80% 目標）
└── recipes/    # 各レシピ種別のエンドツーエンド（"シンプルな避けゲーを作って" 等）
```

## 実行

```
pnpm test            # 全テスト
pnpm test:watch      # ウォッチ
```

## 方針

- コア層は外部 API をモック（LLM 呼び出しは `core/metrics.ts` 越しにスタブ可能）
- レシピテストは実際に worktree を作り、ビルド/テストが通ることを確認
- LLM のコスト懸念から、CI 全開実行は夜間または手動トリガ
